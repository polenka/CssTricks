1. Media Queries - позволяют задать разное оформление страницы для разных типов устройств.

@media only screen and (max-width: 767px)
@media only screen and (max-width: 1279px) and (min-width: 768px)
@media only screen and (max-device-width: 1280px)
@media only screen and (max-width: 940px)
@media only screen and (min-width: 1441px)
@media only screen and (max-device-width: 1280px)
@media screen and (orientation: landscape)
@media screen and (orientation: portrait)
@media only screen and (max-device-width: 800px), only screen and (device-width: 1024px) and (device-height: 600px), only screen and (width: 1280px) and (orientation: landscape), only screen and (device-width: 800px), only screen and (max-width: 767px)

fluid - all and (min-width: 740px) and (min-device-width: 740px), (max-device-width: 800px) and (min-width: 740px) and (orientation:landscape)
normal - all and (min-width: 980px) and (min-device-width: 980px), all and (max-device-width: 1024px) and (min-width: 1024px) and (orientation:landscape)
wide - all and (min-width: 1220px)

stylesheets[all][] = css/icondenver.css

2. Кастомные медиа запросы можно писать в одном файле, можно в разных (так удобнее для ИЕ). Мне нравится подключать файлы с медиа запросами так, чтобы каждая следующая ширина наследовала необходимое из предыдущей, и добавлять новые требуемые свойства для планшетов, средних мониторов, больших мониторов.

3. Принцип "mobile first" - начинаем верстку с мобильной версии. Почему? Чтобы дефолтный вариант, без медиа запросов, работал на старых мобильниках, которые не понимаю @media. В мобильных версиях все в основном располагается в одну колонку, а это естественное поведение потока, таким образом, наращивая дизайн, расставляя элементы на бОльших экранах, можно получить более стройный и чистый css.

4. Библиотеки, которые "учат" ИЕ ниже 9 версии и другие старые браузеры понимать медиа запросы - respond.js или css3-mediaqueries.js

5. Правильный <meta content="width=device-width, initial-scale=1, maximum-scale=1, minimum-scale=1, user-scalable=no" name="viewport">.

6. Responsive css frameworks: http://responsive.gs, http://www.getskeleton.com, http://lessframework.com, http://gumbyframework.com, http://foundation.zurb.com/docs. Source: CSS Frameworks. Или Drupal themes Omega или Zen.

7. Elastic video - http://webdesignerwall.com/tutorials/css-elastic-videos.

8. Elastic images - img {max-width: 100%; height: auto;} (не работает, если изображение находится в блоке с display: inline-block;) Такая конструкция будет работать в IE 7 и IE 9, а для IE 8 делаем такой хак:
@media \0screen { img { width: auto; /* for ie 8 */ } }

10. word-wrap: break-word;

11. Elastic tables - http://css-tricks.com/responsive-data-table-roundup

12. Меню для мобильной версии - http://habrahabr.ru/post/140801

13. Относительные единицы при создании макета. Если, например, в дизайне макет отрисован только на 960px, то для полноэкранного responsive макета это 100%. И все последующие рассчеты ведутся от 960px. Левая колонка - 200px = 20.8333% и т.д.
